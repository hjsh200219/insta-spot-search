## 1. 배경

현재 구조인 “Python의 결정론적 추출 → 에이전트의 판독·검색·검증” 분리는 작고 이해하기 쉽다.
하지만 실제 제품 계약의 핵심인 실행 경로, 외부 콘텐츠 신뢰 경계, 작업 디렉터리 소유권,
인증·오디오 업로드 정책이 Markdown 지침에만 의존한다. 그 결과 기본 실행 경로가 격리된
셸에서 깨지고, 실패한 다운로드가 기존 파일을 삭제하며, 외부 입력이 Bash 가능 에이전트로
직접 전달되는 문제가 확인됐다.

이번 개선의 목적은 기능 확장이 아니라 **현재 기능을 안전하고 재현 가능한 제품 계약으로
고정하는 것**이다.

## 3. 목표

1. 설치된 스킬의 기본 경로가 각 Bash 호출을 독립 셸에서 실행해도 동작한다.
2. 외부 콘텐츠는 항상 데이터로만 취급되고 명령·도구 지시로 승격되지 않는다.
3. 도구가 생성하지 않은 파일이나 디렉터리를 자동 삭제하지 않는다.
4. 브라우저 인증정보 사용과 제3자 오디오 전송은 명시적 선택일 때만 발생한다.
5. `report.json`을 버전 있는 단일 기계 계약으로 만들고 stdout은 표시 전용으로 낮춘다.
6. 네트워크·프로세스·디스크 사용량과 실패가 기존 구조화 exit code 안에서 예측 가능하다.
7. 위 계약을 Python 표준 라이브러리 테스트로 잠근다.

- 전체 오케스트레이션을 Python으로 이전하는 대규모 재작성

## 5. 제품 요구사항

### R1. 격리 셸에서도 안정적인 실행 경로 — P0

- `SKILL_DIR`가 필요한 모든 Bash 블록은 같은 블록 안에서 경로를 다시 계산해야 한다.
- 장기적으로는 반복 Bash 블록 대신 안정적인 단일 래퍼 진입점을 제공할 수 있다.
- 모든 동적 경로와 URL은 따옴표로 감싼다.
- 줌 재추출은 `video.mp4`를 가정하지 않고 `report.json.video_path`를 사용한다.

완료 조건:

- 각 SKILL Bash 블록을 별도 `zsh -c` 프로세스에서 실행하는 계약 테스트가 통과한다.
- 플러그인 루트 경로에 공백·대괄호가 있어도 preflight와 ingest 경로가 올바르게 해석된다.
- MP4, WebM, MKV 입력 모두 실제 `video_path`로 줌 재추출된다.
- 현재 재현 결과인 `/scripts/setup.py` 해석 오류가 다시 발생하지 않는다.

### R2. 작업 디렉터리 소유권과 비파괴적 갱신 — P0

- 자동 생성 작업공간과 사용자가 전달한 `--out-dir`을 구분한다.
- 자동 생성 작업공간에는 소유권 마커와 생성 파일 매니페스트를 기록한다.
- 기존 `video.*`와 프레임은 새 산출물이 성공한 뒤에만 교체한다.
- 정리는 매니페스트에 기록된 파일만 대상으로 하며, 호출자가 제공한 디렉터리 자체는
  재귀 삭제하지 않는다.
- SKILL.md의 범용 `rm -rf <workdir>` 지시를 제거한다.
- 해석된 경로가 작업공간 밖으로 나가거나 심볼릭 링크 경계를 넘으면 정리를 거부한다.

완료 조건:

- 실패한 다운로드가 사전 존재 `video.notes.txt`, `video.mp4`, 사용자 프레임을 보존한다.
- 자동 생성 작업공간만 명시적 cleanup 명령으로 제거할 수 있다.
- 사용자 지정 `--out-dir` cleanup은 도구 생성 파일만 제거한다.
- 저장소 루트, 홈 디렉터리, `/`, 빈 경로를 cleanup 대상으로 전달한 테스트가 모두 거부된다.

### R3. 외부 콘텐츠 불신 경계와 구조화된 조회 — P0

- 캡션, 댓글, OCR 텍스트, 메타데이터, URL, 검색 결과, 내려받은 파일은 모두 **신뢰하지
  않는 데이터**라고 SKILL.md에 명시한다.
- 외부 콘텐츠에 포함된 명령, 정책 변경, 도구 호출, 비밀 읽기 지시는 절대 실행하지 않는다.
- 검색어는 텍스트 값으로만 전달하고 셸 코드나 옵션으로 해석하지 않는다.
- Kakao/Nominatim 호출과 후보 이미지 다운로드는 stdlib 기반 구조화 어댑터 뒤로 이동한다.
- 이미지 다운로드는 HTTPS, 응답 시간 제한, 최대 크기, 이미지 MIME, 작업공간 내부 출력,
  리다이렉트 프로토콜을 검증한다.
- `.claude/settings.json`의 Read 차단을 보안 경계로 과신하지 않는다. Bash를 통한 동일 경로
  접근도 금지한다는 정책을 SKILL에 둔다.

완료 조건:

- 프롬프트 인젝션 문구, 셸 메타문자, 선행 `-` 옵션, 개행을 포함한 캡션/댓글 테스트가
  명령 구조를 바꾸지 않는다.
- 검색과 이미지 조회는 허용된 스킴·크기·시간 제한을 넘으면 안전하게 실패한다.
- 외부 텍스트 때문에 비밀 파일 읽기, 임의 Bash 실행, 작업공간 밖 쓰기가 발생하지 않는다.
- subprocess를 사용할 경우 계속 리스트 인자만 사용하고 `shell=True`는 0건이다.

### R4. 인증·오디오 전송의 명시적 선택 — P1

- `--cookies-browser` 기본값은 `none`으로 변경한다.
- 익명 다운로드 실패 후 쿠키 사용은 사용자가 명시적으로 브라우저를 지정했을 때만 한다.
- 전사는 기본 비활성화하고 `--audio`가 있을 때만 오디오를 추출·업로드한다.
- `--no-audio`는 호환성을 위해 유지하되, 새 기본값과 충돌하지 않도록 deprecation 정책을
  문서화한다.
- 리포트는 익명/쿠키 보조/로컬 입력 여부와 오디오 제공자·업로드 여부를 기록한다.
- 쿠키로만 접근 가능한 콘텐츠를 자동으로 “공개 콘텐츠”로 간주하지 않는다.

완료 조건:

- 기본 실행에서 브라우저 쿠키 접근, 오디오 추출, Whisper API 호출이 0회다.
- `--audio` 없이 환경변수나 `~/.config/watch/.env`에 키가 있어도 외부 업로드하지 않는다.
- 쿠키 보조 접근 시 리포트와 stdout에 그 사실이 표시된다.
- 공개 장소 홍보 콘텐츠가 아님을 확인할 수 없는 인증 전용 소스는 장소 추적을 중단한다.

### R5. 버전 있는 `report.json` 계약 — P1

최소 필드:

- `schema_version`
- `source`, `source_access`(`anonymous`, `cookie-assisted`, `local`)
- `video_path`
- `work_dir`, `work_dir_owned`
- `status`, `warnings`
- `frames`, `frame_count`
- `audio.enabled`, `audio.provider`, `audio.uploaded`, `audio.path`, `audio.transcript`
- 기존 메타데이터·댓글·프로필 필드

추가 규칙:

- stdout은 사람이 읽는 요약이고, 후속 단계는 JSON을 SSOT로 사용한다.
- 외부 원문을 stdout에 표시할 때는 “UNTRUSTED CONTENT” 경계를 명시한다.
- 스키마의 호환성 변경은 `schema_version`을 올리고 SKILL·문서·테스트를 같은 변경에서
  갱신한다.

완료 조건:

- URL과 로컬 파일 입력 모두 필수 필드가 존재하는 스키마 테스트를 통과한다.
- 동적 영상 확장자와 실제 `video_path`가 일치한다.
- 경고성 부분 실패는 `warnings`에 남고 성공으로 숨겨지지 않는다.
- SKILL의 줌·정리 단계가 stdout 문자열이 아닌 JSON 필드를 사용한다.

### R6. 실패·시간·자원 계약 강화 — P1

- 다운로드, probe, 프레임 추출, 오디오 추출, 전사에 단계별 유한 timeout을 둔다.
- `TimeoutExpired`, 파일 생성·삭제·쓰기 실패를 잡아 기존 exit code `2/3/4/5` 중 해당
  단계 코드로 변환한다. 기존 공개 exit code 집합은 유지한다.
- `NaN`, 양·음의 무한대와 비정상적으로 큰 FPS/해상도/프레임 수를 거부한다.
- ffmpeg 성공이어도 프레임이 0장이면 exit 5로 실패한다.
- `--max-frames`는 리포트 목록뿐 아니라 디스크에 남는 프레임 수도 제한한다.
- stdout/stderr에는 키, 쿠키, Authorization 헤더를 출력하지 않는다.

완료 조건:

- 모든 외부 프로세스 호출에 timeout 테스트가 있다.
- 빈 프레임 모의 실행은 exit 5를 반환한다.
- `nan`, `inf`, `-inf`는 argparse 오류로 거부된다.
- 높은 `--fps`에서도 작업공간의 `f_*.jpg`가 `--max-frames`를 넘지 않는다.
- 파일시스템 오류가 traceback/exit 1로 새지 않고 문서화된 코드로 종료한다.

### R7. Whisper 자격증명 argv 노출 제거 — P1

- Bearer 키는 자식 프로세스 argv, 로그, 예외 메시지, 리포트에 포함하지 않는다.
- curl을 유지하면 stdin 기반 config/header 전달을 사용하고, 가능하면 stdlib HTTP 호출로
  전환한다.
- 실패 메시지는 HTTP 상태와 비밀이 제거된 본문 요약만 포함한다.

완료 조건:

- 전사 실행 중 프로세스 명령행 검사에서 API 키가 보이지 않는다.
- 테스트 더블이 받은 argv·stdout·stderr·report JSON 어디에도 테스트 키가 없다.

### R8. stdlib 회귀 테스트와 검증 게이트 — P1

`unittest`로 다음을 자동화한다.

- `parse_ts`, 수치 검증기, 정규식, `load_env_file`, `_brew_pkgs`
- 격리 셸의 SKILL 경로 해석
- 쿠키·오디오 기본값과 명시적 opt-in
- 실패한 다운로드의 기존 파일 보존
- 작업공간 소유권·cleanup 경계
- report schema와 동적 영상 경로
- exit code 매핑, timeout, 빈 프레임
- 외부 콘텐츠 인젝션·URL·경로 이스케이프
- `--comments 0` 명령 구성

`scripts/verify-docs.py`는 다음까지 확장한다.

- 저장소 내부 Markdown 링크 실재 여부
- SKILL Bash 블록의 독립 셸 실행 가능성 또는 정적 경로 초기화 규칙
- 문서화된 exit code와 실제 분기의 의미 정합성
- `shell=True`, 범용 `rm -rf`, 비밀 헤더 argv 패턴 금지

완료 조건:

- `python3 -m unittest discover`와 `python3 scripts/verify-docs.py`가 모두 exit 0이다.
- 네트워크, 실제 브라우저 쿠키, 실제 API 키 없이 테스트가 재현된다.
- 새 pip 의존성 및 의존성 매니페스트는 0개다.

### R9. 소규모 정합성 개선 — P2

- `attempts` 타입과 `last`의 비-옵셔널 흐름을 명확히 해 현재 mypy 오류 4건을 제거한다.
- `--comments 0`일 때 yt-dlp에 `--no-write-comments`를 전달한다.
- 아키텍처 문서의 “6-layer”와 0~6 레이어 표기를 하나의 용어로 통일한다.
- setup의 Homebrew 자동 설치가 조회 전용 경계 밖의 호스트 변경임을 명시하고, 설치 전
